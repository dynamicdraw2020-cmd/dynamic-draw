Supabase에 데이터베이스 설치

회원과 결과를 저장하는 전자 노트를 만드는 단계입니다.

내 컴퓨터에서 누를 것

  1. 압축 푼 폴더에서 같은 폴더의 1_SUPABASE에_한번만_붙여넣기.sql에 마우스 오른쪽 버튼을 누릅니다.
  2. 연결 프로그램 → 메모장을 누릅니다.
  3. Ctrl + A, 그다음 Ctrl + C를 누릅니다.
DB 설치 SQL 파일 열기

Supabase에서 누를 것

  1. Supabase 로그인 → dynamic-draw 프로젝트 카드를 누릅니다.
  2. 왼쪽의 SQL Editor를 누릅니다.
  3. 위쪽의 New query를 누릅니다.
  4. 큰 빈칸을 누르고 Ctrl + V를 누릅니다.
  5. 위쪽 또는 오른쪽 아래 초록색 Run을 누릅니다.
  6. Success가 나오면 끝입니다.
한 번만 실행
새 Supabase 프로젝트에서 처음 한 번만 실행하세요. 빨간 오류가 나오면 같은 버튼을 계속 누르지 말고 오류 문장을 확인하세요.

GitHub에 홈페이지 파일 올리기

ZIP 한 개가 아니라 압축을 푼 안쪽 파일들을 올립니다.

  1. GitHub 로그인 → dynamic-draw 저장소를 누릅니다.
  2. 파일이 있으면 Add file → Upload files를 누릅니다. 비어 있으면 uploading an existing file을 누릅니다.
  3. 컴퓨터의 압축 푼 폴더 안에서 Ctrl + A로 모두 선택합니다.
  4. GitHub의 Drag files here 영역으로 끌어다 놓습니다.
  5. 업로드가 끝나면 아래의 초록색 Commit changes를 누릅니다.
  6. GitHub 첫 화면에 package.json이 바로 보이면 성공입니다.
올리면 안 되는 것
Secret key, 비밀번호 메모, .env.local, node_modules, .next

Supabase 열쇠 3개 복사

Supabase 프로젝트의 Connect 또는 Project Settings → API Keys에서 찾습니다.

  1. Supabase 프로젝트 위쪽의 Connect를 누릅니다.
  2. 안 보이면 왼쪽 아래 Project Settings → API Keys를 누릅니다.
  3. Project URL, Publishable key, Secret key를 메모장에 잠깐 복사합니다.
Secret key는 공개 금지
GitHub·채팅·브라우저 파일에 넣지 말고 Vercel 환경변수에만 저장합니다.

Vercel에 연결하고 환경변수 넣기

Vercel이 GitHub 코드를 실제 인터넷 홈페이지로 켭니다.

  1. Vercel 로그인 → 오른쪽 위 Add New... → Project.
  2. GitHub의 dynamic-draw 오른쪽 Import.
  3. Framework가 Next.js인지 확인.
  4. Environment Variables를 찾습니다.

아래 6개를 한 줄씩 추가

NEXT_PUBLIC_SUPABASE_URLSupabase Project URL
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEYPublishable key 또는 anon key
SUPABASE_SECRET_KEYSecret key 또는 service_role key
NEXT_PUBLIC_SITE_URLhttps://dynamic2020.com
NEXT_PUBLIC_DEMO_MODEfalse
ADMIN_SETUP_SECRET직접 만든 32자 이상의 긴 비밀문자
  1. 각 줄에서 Name과 Value를 넣고 Add를 누릅니다.
  2. 6개가 모두 보이면 Deploy를 누릅니다.
  3. 축하 화면에서 Visit을 누릅니다.
ADMIN_SETUP_SECRET 예시 모양
DynamicDraw-Setup-2026-My-Private-Key-93841
예시를 그대로 쓰지 말고 본인만 아는 값으로 바꾸세요.

Supabase 로그인 돌아갈 주소 설정

이메일 인증 뒤 회원이 홈페이지로 돌아오게 합니다.

  1. Supabase → Authentication → URL Configuration.
  2. Site URL에 https://dynamic2020.com 입력.
  3. Redirect URLs에 아래 3개를 추가.
  4. Save.

dynamic2020.com 연결

Vercel이 보여주는 값을 가비아에 그대로 옮깁니다.

Vercel에서

  1. 프로젝트 → Settings.
  2. Domains.
  3. Add Domain.
  4. dynamic2020.com 입력 후 Add.
  5. www.dynamic2020.com도 추가.
  6. 화면의 A와 CNAME 값을 열어 둡니다.

가비아에서

  1. My가비아 → 서비스 관리 → DNS 관리툴.
  2. dynamic2020.com 체크 → DNS 설정.
  3. + 레코드 추가.
  4. A / 호스트 @ / Vercel의 A 값.
  5. CNAME / 호스트 www / Vercel의 CNAME 값.
  6. 각 줄 확인 → 마지막 저장.
숫자를 외우지 마세요
이 문서의 예시보다 Vercel Domains 화면에 지금 표시되는 값을 그대로 복사하는 것이 정확합니다.
9

진짜 작동하는지 시험

아래 순서가 모두 되면 실제 운영 준비가 끝난 것입니다.

  1. 다른 브라우저에서 일반 회원가입 신청.
  2. 관리자 로그인 → 회원 관리.
  3. 승인·ID 자동발급.
  4. 관리자 → 실시간 추첨에서 회원 선택.
  5. 다른 휴대전화에서 /live 열기.
  6. 추첨 실행 → 흔들림 → 빛 → 뒤집기 → 결과 확인.
  7. 회원 로그인 → 내 정보에서 상품 수량 확인.
  8. 회원 직접 교환 또는 관리자 교환 시스템 → 관리자 현장 교환 시험.
  9. 확률 변경 기록과 관리자 로그 확인.
