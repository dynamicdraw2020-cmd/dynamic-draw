Dynamic Draw v1.0.2 — 이미 배포한 사람용 수정 순서
=====================================================

처음부터 다시 할 필요 없습니다.
GitHub 저장소, Vercel 프로젝트, Supabase 프로젝트를 삭제하지 마세요.

지금 보이는 오류의 원인
----------------------
기존 DB 설치 SQL에 service_role의 표 접근 권한이 빠져 있었습니다.
그래서 공개 홈페이지는 보이지만 /setup-admin의 서버용 조회만 실패했습니다.

가장 먼저 할 일 — 1분
---------------------
1) 이 폴더의 "4_기존설치_권한오류_수정.sql"을 메모장으로 엽니다.
2) Ctrl+A → Ctrl+C.
3) Supabase → SQL Editor → New query.
4) 큰 칸에 Ctrl+V.
5) 오른쪽 아래 초록색 Run.
6) 결과에서 ready=true, serviceRoleCanReadProfiles=true가 보이면 성공.
7) 기존 Vercel 주소의 /setup-admin을 새로고침합니다.
8) 설치용 비밀문자, 관리자 이름, 이메일, 비밀번호를 넣어 관리자를 만듭니다.

이 단계만으로 현재 배포본의 관리자 생성 오류가 바로 고쳐집니다.

코드도 수정판으로 교체하는 이유
------------------------------
- 잘못된 URL에 /rest/v1을 붙여 넣어도 자동 정리
- 새 Secret key와 레거시 service_role key 둘 다 지원
- DB 표 없음 / 권한 없음 / Secret key 오류를 서로 구분해서 표시
- /api/setup-admin/status 진단 주소 추가
- 새 설치 SQL에는 service_role 권한이 처음부터 포함됨

GitHub 코드 교체
---------------
전체 수정본 ZIP의 압축을 푼 뒤 GitHub 저장소에서 Add file → Upload files로
같은 경로의 파일을 덮어쓰고 Commit changes를 누릅니다.
Vercel은 GitHub 변경을 감지해 자동으로 다시 배포합니다.

비밀값은 절대 보내거나 GitHub에 올리지 마세요.
.env, .env.local, Secret key, ADMIN_SETUP_SECRET 값은 업로드 금지입니다.
