Dynamic D v1.4.0 적용 방법

이번 업데이트는 두 가지를 해야 합니다.
1) Supabase SQL 실행
2) GitHub 파일 교체

기존 회원, 관리자, 도메인, 환경변수는 건드리지 않습니다.

[1] Supabase SQL 실행
1. Dynamic-Draw-v1.4.0-DynamicD_공개개편_전체추첨_DB.sql 파일을 엽니다.
2. Ctrl + A
3. Ctrl + C
4. Supabase 접속
5. SQL Editor 클릭
6. New query 클릭
7. 붙여넣기
8. Run 클릭
9. 결과가 아래처럼 true면 성공입니다.
   raffle_events_ready = true
   raffle_function_ready = true
   dynamic_d_settings_ready = true

[2] GitHub 파일 교체
1. Dynamic-Draw-GitHub-교체파일-v1.4.0.zip 압축을 풉니다.
2. GitHub dynamic-draw 저장소로 갑니다.
3. Add file 클릭
4. Upload files 클릭
5. 압축 푼 폴더 안쪽 파일 전부를 드래그합니다.
6. Commit changes 클릭
7. Vercel 자동 배포가 끝날 때까지 기다립니다.
8. Vercel Deployments에서 Ready 확인
9. 홈페이지에서 Ctrl + F5

[3] 적용 후 확인
- 첫 화면 서비스명이 Dynamic D로 보이는지 확인
- 첫 화면에 공지와 이벤트가 보이는지 확인
- /raffles 페이지가 열리는지 확인
- 관리자 → 전체 회원 추첨 메뉴가 보이는지 확인
- 전체 회원 추첨 만들기 → 전체 회원 추첨 시작이 되는지 확인

문제가 생기면 Vercel Logs 화면 캡처를 보내면 됩니다.
