Dynamic D v1.6.0 GitHub 교체용 파일

적용 순서
1. Supabase SQL Editor에서 DynamicD-v1.6.0-SQL_교체용_메모장.txt 내용을 먼저 실행합니다.
2. 이 압축 파일의 app / components / lib 폴더를 GitHub 저장소 루트에 같은 경로로 덮어씁니다.
3. GitHub에 커밋하면 Vercel이 자동 재배포됩니다.
4. 배포 완료 후 /admin/secret-codes 에서 시크릿코드를 발급하고 /signup 에서 가입을 테스트합니다.

주의
- 기존 기능 삭제가 아니라, 같은 경로 파일을 교체/추가하는 패치입니다.
- 시크릿코드는 발급 후 4시간만 유효하며 1회 사용 후 재사용할 수 없습니다.
- CS_MANAGER는 회원 승인/정지/정지해제, 문의 답변, 개별 추첨권/포인트 지급만 허용됩니다.
