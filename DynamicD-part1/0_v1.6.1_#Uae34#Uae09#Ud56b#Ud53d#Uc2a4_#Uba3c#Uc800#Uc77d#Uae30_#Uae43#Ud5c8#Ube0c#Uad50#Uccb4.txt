Dynamic D v1.6.1 긴급 핫픽스 GitHub 교체용 파일

이번 핫픽스 목적
1. v1.6.0 적용 후 Vercel 빌드/로딩이 멈출 수 있는 회원가입 라우트 import 오류를 수정합니다.
2. 회원가입 시크릿코드 기능을 다시 연결합니다.
3. CS_MANAGER 권한을 실제 페이지/API에 반영합니다.
4. 관리자 페이지에 서버 상태 화면을 추가합니다.
5. /api/ping과 트래픽 체크 스크립트를 추가합니다.
6. 빈 화면/무한 로딩 대신 오류 화면과 로딩 안내를 표시합니다.

적용 순서
1. Supabase SQL Editor에서 DynamicD-v1.6.1-SQL_교체용_메모장.txt 전체를 실행합니다.
2. 이 압축 파일 안의 app / components / lib / tools 폴더를 GitHub 저장소 루트에 같은 경로로 덮어씁니다.
3. GitHub 커밋 후 Vercel 배포가 Ready가 될 때까지 기다립니다.
4. https://배포주소/api/ping 을 엽니다. ok:true가 나오면 앱 라우트는 살아 있습니다.
5. 관리자로 로그인 후 /admin/server-status 를 확인합니다.
6. /admin/secret-codes 에서 코드를 발급하고 /signup에서 가입 테스트를 진행합니다.

트래픽 검증
로컬 PC에서 저장소를 받은 뒤 아래처럼 실행합니다.
node tools/dynamicd-traffic-check.mjs https://배포주소 20 200 /api/ping

처음부터 높은 수치로 때리지 말고 10 → 20 → 50 순서로 올리세요.
