Dynamic Draw v1.0.3 검증 결과
============================
검증일: 2026-06-25

1. 현재 GitHub 상태 재현 테스트
- 기존 Dynamic-Draw-FINAL에 사용자가 올린 v1.0.2 교체 파일을 덮어써 현재 상태를 재현함.
- 기존 오류 재현: lib/setup-status.ts가 envDiagnostics를 가져오지만 구버전 lib/env.ts에는 해당 export가 없어 TypeScript 실패.
- v1.0.3 교체 파일을 같은 상태에 덮어쓴 뒤 아래 검사를 다시 실행함.

2. 패키지 설치
- npm ci --no-audit --no-fund --prefer-offline
- 결과: 성공 (403 packages)

3. TypeScript
- npm run typecheck
- 결과: 성공

4. ESLint
- npm run lint
- 결과: 성공

5. Next.js 프로덕션 빌드
- NEXT_TELEMETRY_DISABLED=1 CI=1 npm run build
- 결과: 성공 / exit code 0
- Next.js: 16.2.9
- 공개·회원·관리자·API 경로가 빌드 출력에 정상 등록됨.

6. 프로덕션 서버 기본 화면 확인
- npm start, PORT=3104
- /              → HTTP 200
- /signup        → HTTP 200
- /setup-admin   → HTTP 200
- /api/health    → HTTP 200 (환경변수 없는 미리보기 상태)

7. SQL 정적 구문 검사
- PostgreSQL parser(pglast)로 아래 SQL 파싱 성공
  · 4_DB_보정_v1.0.3.sql
  · supabase/PASTE_THIS_ONCE.sql
  · 1_SUPABASE에_한번만_붙여넣기.sql
  · 초기 스키마 migration
  · v1.0.3 보정 migration

8. 수정 내용
- env.ts와 setup-status.ts 버전 일치
- envDiagnostics export 포함
- pgcrypto 스키마 위치를 런타임에 찾아 호출하는 내부 래퍼 추가
- 해시 체인 검증 함수를 실제 검증 로직으로 복원
- 보안 난수 추첨 함수 연결 보정
- Rate Limit DB 오류와 실제 과다 요청을 구분
- 회원가입을 관리자 승인형 서버 생성 방식으로 안정화
- 회원가입 오류 원인별 안내 추가

주의
- 사용자의 Supabase Secret key는 전달받지 않았으므로 실제 운영 DB에 대한 원격 통합 테스트는 수행하지 않음.
- 따라서 적용 후 /api/health, 회원가입 1건, 승인 1건, 추첨 1건을 운영 환경에서 최종 확인해야 함.
- v1.0.3 SQL은 사용자·회원·결과·설정 데이터를 삭제하지 않으며 rate_limits의 임시 제한 기록만 초기화함.
