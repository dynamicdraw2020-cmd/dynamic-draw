Dynamic D v1.0.1 퇴근 패치 적용 방법

이번 패치는 처음부터 다시 하는 게 아닙니다.
Supabase SQL 1번 실행 + GitHub 파일 교체만 하면 됩니다.
환경변수, 도메인, 관리자 계정은 건드리지 마세요.

1) Supabase SQL 실행
- Dynamic-D-v1.0.1-퇴근패치_DB.sql 파일을 메모장으로 엽니다.
- Ctrl + A → Ctrl + C
- Supabase → SQL Editor → New query
- Ctrl + V
- Run 클릭
- 아래 값들이 true로 나오면 성공입니다.
  brand_text_ready
  version_fixed
  raffle_live_ready
  default_notices_removed

2) GitHub 파일 교체
- Dynamic-D-GitHub-교체파일-v1.0.1-퇴근패치.zip 압축을 풉니다.
- GitHub dynamic-draw 저장소로 갑니다.
- Add file → Upload files
- 압축 푼 폴더 안쪽 파일 전부 드래그합니다.
- Commit changes 클릭
- Vercel 자동 배포가 Ready가 될 때까지 기다립니다.

3) 확인
- dynamic2020.com에서 Ctrl + F5
- 메인 문구가 𝐃𝐲𝐧𝐚𝐦𝐢𝐜 𝐃 / 𝐃𝐲𝐧𝐚𝐦𝐢𝐜 Event server로 보이는지 확인
- event-live 페이지가 실시간 추첨에 반응하는지 확인
- 관리자 뽑기 관리에서 상품 추가 시 reset 오류가 사라졌는지 확인
- 추첨권·화폐 관리에서 새로 만든 뽑기가 선택 목록에 보이는지 확인

주의:
기본 테스트 공지는 SQL에서 제거됩니다.
사용자가 직접 만든 공지는 제목이 기본값과 정확히 일치하지 않으면 삭제하지 않습니다.
