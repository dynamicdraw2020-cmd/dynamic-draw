Dynamic Draw v1.0.3 적용 방법
=============================

중요: GitHub·Vercel·Supabase를 처음부터 다시 만들지 않습니다.
기존 관리자·회원·추첨 결과·도메인·환경변수도 그대로 유지됩니다.

1단계 — Supabase DB 보정
-----------------------
1. 이 폴더의 "4_DB_보정_v1.0.3.sql"을 메모장으로 엽니다.
2. Ctrl+A → Ctrl+C로 전체 복사합니다.
3. Supabase → SQL Editor → New query를 누릅니다.
4. 붙여넣고 초록색 Run을 누릅니다.
5. 마지막 결과에서 아래 항목을 확인합니다.
   patch_applied = true
   ready = true
   digest_ready = true
   random_ready = true
6. admin_log_chain과 probability_history_chain에는 JSON 결과가 표시됩니다.
   valid가 true면 정상입니다.

2단계 — GitHub 코드 교체
-----------------------
1. "Dynamic-Draw-GitHub-교체파일-v1.0.3.zip"의 압축을 풉니다.
2. GitHub → dynamic-draw 저장소로 들어갑니다.
3. Add file → Upload files를 누릅니다.
4. 압축을 푼 폴더 안쪽 파일을 모두 선택해 업로드합니다.
   ZIP 파일 자체를 올리면 안 됩니다.
5. 아래 초록색 Commit changes를 누릅니다.
6. 같은 이름의 기존 파일은 자동으로 새 파일로 교체됩니다.

3단계 — Vercel 확인
------------------
1. Vercel → dynamic-draw → Deployments로 이동합니다.
2. 가장 위 배포가 Building에서 Ready로 바뀔 때까지 기다립니다.
3. Ready가 되면 Visit을 누릅니다.
4. 환경변수는 다시 입력하지 않습니다.
5. 도메인도 다시 연결하지 않습니다.

4단계 — 동작 확인
----------------
1. 사이트 주소 뒤에 /api/health를 붙여 엽니다.
2. status가 healthy이면 DB 연결 정상입니다.
3. 회원가입 신청을 시험합니다.
4. 관리자로 로그인 → 회원 관리 → 승인·ID 자동발급을 누릅니다.
5. 관리자 → 실시간 추첨 → 뽑기와 회원 선택 → 추첨 실행을 누릅니다.

뽑기 순서
---------
일반 회원이 가입 신청
→ 관리자가 회원 관리에서 승인
→ 고유 ID 자동 발급
→ 관리자 메뉴의 실시간 추첨
→ 진행 중인 뽑기와 승인 회원 선택
→ 추첨 실행
→ 카드 애니메이션 뒤 결과 공개
→ 교환 가능한 상품은 회원 보관함에 자동 적립

v1.0.3에서 수정한 내용
----------------------
- Vercel 빌드 오류: envDiagnostics 내보내기 누락 해결
- setup-admin 파일 버전 불일치 해결
- pgcrypto 설치 위치와 무관하게 해시·보안 난수 함수 사용
- 임시로 단순화됐던 감사 로그 검증 함수 원상 복구
- Rate Limit DB 오류를 "요청이 많음"으로 잘못 표시하던 문제 해결
- 회원가입을 관리자 승인형 흐름에 맞게 서버에서 안정적으로 생성
- 회원가입 실패 시 실제 원인에 맞는 안내 문구 표시
