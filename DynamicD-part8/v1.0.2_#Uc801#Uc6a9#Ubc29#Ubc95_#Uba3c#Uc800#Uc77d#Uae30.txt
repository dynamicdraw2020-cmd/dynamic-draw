Dynamic Draw v1.0.2 적용 방법
==============================

이번 수정은 기존 사이트를 지우거나 처음부터 다시 만드는 작업이 아닙니다.
현재 GitHub, Vercel, Supabase, 관리자 계정과 데이터는 그대로 유지됩니다.

[가장 먼저: 현재 오류 즉시 고치기]
1. 이 폴더의 4_PGCRYPTO_오류_수정.sql 파일을 엽니다.
2. Ctrl+A -> Ctrl+C로 전부 복사합니다.
3. Supabase -> SQL Editor -> New query를 누릅니다.
4. 빈 입력칸에 붙여넣고 오른쪽 아래 Run을 누릅니다.
5. 맨 아래 결과에서 다음 네 값이 true인지 확인합니다.
   ready
   pgcrypto_installed
   hash_functions_fixed
   random_function_fixed
6. 기존 Dynamic Draw 화면으로 돌아가 Ctrl+F5를 누릅니다.
7. 오류가 났던 작업을 다시 실행합니다.

주의: 1_SUPABASE에_한번만_붙여넣기.sql 전체를 다시 실행할 필요가 없습니다.

[GitHub 코드도 v1.0.2로 영구 교체하기]
1. Dynamic-Draw-GitHub-교체파일-v1.0.2.zip의 압축을 풉니다.
2. GitHub dynamic-draw 저장소에서 Add file -> Upload files를 누릅니다.
3. 압축을 푼 안쪽 파일과 폴더를 모두 끌어다 놓습니다.
4. Commit changes를 누릅니다.
5. Vercel이 자동으로 새 배포를 시작합니다.
6. Vercel의 Deployments에서 가장 위 배포가 Ready가 될 때까지 기다립니다.
7. https://본인주소/api/health 를 열어 아래처럼 보이는지 확인합니다.
   status: healthy
   schemaVersion: 1.0.2
   pgcryptoReady: true

환경변수는 다시 입력하지 않습니다.
도메인도 다시 연결하지 않습니다.
Supabase 프로젝트도 새로 만들지 않습니다.

[관리자 계정]
이미 최고 관리자를 만들었다면 /setup-admin을 다시 사용할 필요가 없습니다.
로그인 이메일은 Supabase SQL Editor에서 아래 코드로 확인할 수 있습니다.

select email, display_name, role, status
from public.profiles
where role = 'SUPER_ADMIN'
order by created_at desc;

비밀번호 원문은 DB에서 확인할 수 없습니다. 잊었다면 Supabase Authentication -> Users에서
해당 이메일 계정의 비밀번호 재설정 절차를 사용합니다.
