## 수정한 핵심

### 1. 스탭업 설정을 더 쉽게 변경

`/admin/step-events`에 프리셋 기반 빠른 생성 폼을 추가했습니다.

추가된 프리셋:

- 신규가입 1STEP
- 로그인 3일
- 출석 7일
- 쿠폰 사용 1회
- 관리자 수동 지급

이제 처음 설정할 때 이벤트를 만들고, 다시 STEP을 만들고, 보상을 연결하는 과정을 한 번에 시작할 수 있습니다.

### 2. 사용법 설명 강화

관리자 화면에 다음 안내를 추가했습니다.

- 스탭업 기본 사용 순서
- DRAFT / ACTIVE / PAUSED / ENDED / ARCHIVED 의미
- ONCE / DAILY / WEEKLY / SEASON 반복 기준 의미
- 미션 종류별 자동 반영 여부
- 보상 타입별 필요한 대상 선택 안내

### 3. 보상 설정 UI 개선

기존에는 보상 타입을 골라도 포인트, 뽑기, 상품, 랜덤박스, 쿠폰 선택창이 전부 보여서 헷갈렸습니다.

이제는 보상 타입에 따라 필요한 선택창만 보입니다.

예:

- 포인트/화폐 선택 시 화폐 선택창만 표시
- 뽑기권 선택 시 뽑기 선택창만 표시
- 랜덤박스 선택 시 랜덤박스 선택창만 표시
- 쿠폰 선택 시 쿠폰 선택창만 표시
- EXP 선택 시 추가 대상 선택 없음

### 4. 잘못된 보상 저장 방지

보상 타입이 포인트/뽑기권/상품/랜덤박스/쿠폰인데 대상을 선택하지 않은 경우, 저장 단계에서 명확한 오류가 나오도록 막았습니다.

예:

- 포인트/화폐 보상은 지급할 화폐를 선택해야 합니다.
- 뽑기권 보상은 지급할 뽑기를 선택해야 합니다.
- 쿠폰 보상은 지급할 쿠폰을 선택해야 합니다.

### 5. STEP 수정 시 기존 보상 보존

기존에는 STEP 수정 화면에서 보상 입력값이 제대로 보이지 않아 수정 저장 시 기존 보상이 다른 기본값으로 덮일 위험이 있었습니다.

이제 STEP 수정 화면에 기존 보상 타입, 수량, 대상, 메모, 추가 보상 JSON이 채워지도록 고쳤습니다.

### 6. 관리자 수동 진행/테스트 도구 추가

`/admin/step-events` 하단에 수동 진행 도구를 추가했습니다.

용도:

- 자동 연동 전 테스트
- 관리자 확인형 미션 운영
- 특정 회원의 진행도 수동 증가

회원 선택 → 미션 종류 선택 → 증가량 입력 → 진행도 반영 흐름입니다.

## 자동 반영되는 미션

현재 코드에서 자동으로 진행도가 올라가는 미션은 다음입니다.

- 회원가입
- 로그인
- 출석
- 랜덤박스 오픈
- 쿠폰 사용
- 관리자 수동 지급

아래 미션들은 목록에는 있지만, 별도 API 연결이 추가되기 전까지는 관리자 수동 진행으로 운영하는 것이 안전합니다.

- 게시글 작성
- 댓글 작성
- 좋아요
- 포인트 획득
- 포인트 사용
- 이벤트 참여
- 후원
- 친구 초대
- 기타

## 운영 추천 순서

1. `/admin/step-events` 접속
2. 프리셋으로 이벤트 + 첫 STEP 생성
3. 이벤트 목록에서 STEP 관리 확인
4. 테스트 회원으로 수동 진행 도구 실행
5. 유저 화면 `/step-events`에서 진행도 확인
6. 문제 없으면 이벤트를 `ACTIVE`로 전환
