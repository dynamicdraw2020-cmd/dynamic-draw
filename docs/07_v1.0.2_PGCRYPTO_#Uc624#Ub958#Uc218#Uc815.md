# Dynamic Draw v1.0.2 — pgcrypto 오류 수정

## 수정 대상

다음 오류가 발생한 기존 Supabase 프로젝트입니다.

```text
function digest(text, unknown) does not exist
```

또는 추첨 실행 시 다음 함수 조회 오류가 발생하는 경우도 포함합니다.

```text
function gen_random_bytes(integer) does not exist
```

## 원인

Supabase의 PostgreSQL 확장은 일반적으로 `extensions` 스키마에 설치됩니다. 기존 Dynamic Draw 보안 함수는 실행 시 검색 경로를 `public`으로만 고정해 두어 `pgcrypto`의 `digest()`와 `gen_random_bytes()`를 찾지 못했습니다.

## 즉시 수정

루트의 `4_PGCRYPTO_오류_수정.sql`을 Supabase SQL Editor에서 한 번 실행합니다. 이 패치는 다음 작업만 수행합니다.

- 실제 `pgcrypto` 설치 스키마 확인
- SHA-256 및 보안 난수 함수 자체 시험
- 영향을 받는 Dynamic Draw 함수 5개의 안전한 검색 경로 수정
- 설치 진단 버전을 `1.0.2`로 갱신
- PostgREST 스키마 캐시 새로고침

회원, 관리자, 추첨 결과, 상품, 교환 기록, 감사 로그는 삭제하지 않습니다. 패치는 여러 번 실행해도 안전합니다.

## 수정한 DB 함수

- `append_admin_log`
- `admin_update_probabilities`
- `execute_draw`
- `verify_admin_log_chain`
- `verify_probability_history_chain`

## 코드 수정

새 설치 SQL은 `pg_catalog, extensions, public` 검색 경로를 사용하며 전체 설치 SQL의 RLS 정책 생성도 다시 실행할 수 있도록 보완했습니다. API는 동일한 DB 오류가 발생할 경우 원시 PostgreSQL 문구 대신 실행할 수정 파일을 안내합니다.
