Dynamic Draw v1.0.4 적용 방법

이번 수정은 프론트엔드 코드만 바꾸는 업데이트입니다.
Supabase SQL, 환경변수, 도메인, 관리자 계정은 건드리지 않습니다.

수정 내용
- 뽑기 생성 성공 후 reset 오류 수정
- 상품 추가 성공 후 reset 오류 수정
- 교환 규칙 생성 성공 후 reset 오류 수정
- 회원가입 성공 후 reset 오류 수정

적용 순서
1. Dynamic-Draw-GitHub-교체파일-v1.0.4.zip을 압축 풉니다.
2. GitHub의 dynamic-draw 저장소를 엽니다.
3. Add file → Upload files를 누릅니다.
4. 압축을 푼 폴더 안쪽의 components 폴더, package.json, package-lock.json을 업로드합니다.
5. 기존 파일을 덮어쓰는 상태로 Commit changes를 누릅니다.
6. Vercel이 자동 배포할 때까지 기다립니다.
7. Vercel Deployments에서 최신 배포가 Ready인지 확인합니다.
8. 홈페이지를 Ctrl+F5로 새로고침합니다.

주의
- 기존 GitHub 저장소를 삭제하지 마세요.
- Supabase SQL을 다시 실행하지 마세요.
- .env 또는 Secret key를 GitHub에 올리지 마세요.
- 오류 팝업이 뜨기 전에 뽑기 생성 API는 이미 성공했을 수 있습니다. 관리자 뽑기 목록을 새로고침해 같은 이름의 뽑기가 이미 생겼는지 먼저 확인하세요.
