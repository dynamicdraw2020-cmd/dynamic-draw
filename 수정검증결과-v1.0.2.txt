Dynamic Draw v1.0.2 수정 및 검증 결과
검증일: 2026-06-25

[확인한 원인]
- 기존 보안 함수는 SECURITY DEFINER 실행 경로를 public으로만 고정했습니다.
- Supabase의 pgcrypto는 일반적으로 extensions 스키마에 설치됩니다.
- 따라서 append_admin_log 등의 함수가 digest()와 gen_random_bytes()를 찾지 못해
  "function digest(text, unknown) does not exist" 오류가 발생했습니다.

[수정 사항]
- 기존 DB를 삭제하지 않는 4_PGCRYPTO_오류_수정.sql 추가
- 실제 pgcrypto 설치 스키마를 DB에서 찾아 보안 함수 5개의 search_path를 자동 수정
- SHA-256 함수와 보안 난수 함수 자체 시험 포함
- 신규 설치 SQL의 검색 경로를 pg_catalog, extensions, public으로 수정
- 신규 설치 SQL의 RLS 정책을 재실행 가능하게 보완(drop policy if exists)
- 설치 상태 schemaVersion을 1.0.2로 갱신하고 pgcryptoReady 진단 추가
- /api/health에서 DB 버전, pgcrypto, service_role 조회 상태 확인 가능
- RPC 오류 발생 시 원시 PostgreSQL 문구 대신 실행해야 할 수정 SQL 안내

[실제 Supabase에서 사용자가 확인할 항목]
- 4_PGCRYPTO_오류_수정.sql 실행 마지막 결과에서 다음 값이 모두 true여야 합니다.
  ready
  pgcrypto_installed
  hash_functions_fixed
  random_function_fixed
- v1.0.2 코드 배포 뒤 /api/health에서 status=healthy,
  schemaVersion=1.0.2, pgcryptoReady=true를 확인합니다.

[중요]
- 사용자의 실제 Supabase 비밀키와 DB 접속 권한은 전달받지 않았으므로
  실제 운영 DB에는 이 환경에서 접속하거나 변경하지 않았습니다.
- 패치 SQL은 회원, 관리자, 상품, 결과, 교환 기록과 설정을 삭제하지 않습니다.
- GitHub, Vercel, Supabase와 도메인을 처음부터 다시 만들 필요가 없습니다.
